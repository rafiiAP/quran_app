🎯 Priority Improvements

🔴 High Priority

Remove Dio imports from repositories — SurahRepositoryImpl and JadwalSholatRepositoryImpl import package:dio/dio.dart to catch DioException. This should be handled at datasource level. Datasources should throw typed exceptions (e.g., NetworkException), and repositories should only catch those.

🟠 Medium Priority

Split JadwalSholatPageCubit (419 lines) — Extract to: LocationService (abstract GPS/geocoding), CountdownHelper (static timer logic), and keep notification toggle in the cubit. This improves testability and SRP.

🟠 Medium Priority

Use int.tryParse for prayer time parsing — jadwal_sholat_page_cubit.dart:parseJadwal() uses int.parse(data.fajr.split(':')[0]) which throws on malformed API data. Replace with int.tryParse + fallback.

🟠 Medium Priority

Split SurahRepository — Currently handles both surah list and detail surah. Move getDetailSurah() to a dedicated DetailSurahRepository in the detail_surah feature.

🟡 Low Priority

Extract sub-widgets from god pages — detail_surah_page.dart (392 lines) and home_page.dart (326 lines) would benefit from extracting reusable sections into separate widget classes.

🟡 Low Priority

Replace placeholder certificate pins — CertificatePins contains placeholder SHA-256 hashes. Extract real pins from production certificates before release.

🟡 Low Priority

Update project-context documentation — States "dartz" but codebase uses fpdart. Sync documentation with reality.
