💪 Key Strengths

  • Clean Architecture yang solid — separation domain/data/presentation sangat baik, dependency rule terjaga (domain tidak depend ke data/framework)
  • BLoC Cubit + Freezed — state management konsisten, predictable, dan testable; migrasi dari GetX 100% sukses
  • Functional error handling — Either<Failure, T> menghilangkan exception-based control flow di business logic
  • Test coverage tinggi (94.5%) — comprehensive unit test termasuk property-based test
  • Abstraction layer yang tepat — AppHttpClient, CrashReporter, NotificationService, LocalStorageService semuanya mockable

🔧 Areas for Improvement

High Priority

  • Pecah RemoteUsecase menjadi per-feature UseCase — GetSurahUseCase, GetDetailSurahUseCase, GetJadwalSholatUseCase. Ini penting untuk SRP dan scalability.
  • Refactor God Mixin (C/W) — Pecah menjadi service kecil yang di-inject via constructor, bukan global accessor. Ini bottleneck terbesar untuk testability widget layer.
  • Scoped BlocProvider — Pindahkan Cubit yang hanya dipakai di 1 halaman (DetailSurahCubit, JadwalSholatPageCubit) ke scoped provider di route-level.

Medium Priority

  • Widget/Integration tests — Tambah minimal happy-path widget test untuk setiap page utama
  • Migrasi dartz → fpdart — dartz sudah unmaintained, fpdart aktif dan Dart 3 compatible
  • Feature-based folder restructure — Pertimbangkan modularisasi per-feature jika app terus berkembang
  • Input validation di Model.fromMap — Handle null/malformed JSON gracefully

Low Priority

  • Tambahkan stricter analysis_options.yaml (recommended lints dari very_good_analysis atau custom)
  • CI pipeline yang lebih lengkap (flutter analyze, flutter test, coverage gate)
  • Tambah certificate pinning untuk production
  • Pertimbangkan Riverpod atau injectable untuk DI yang lebih type-safe
