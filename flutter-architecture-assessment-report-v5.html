1. Layer Separation & Dependency Rule

10/10
Domain layer imports only fpdart, equatable, freezed_annotation — zero violations
Repository files have zero Dio imports — DataException abstraction encapsulates network
Data never imports presentation. Explicit toEntity() mapping in all Models

2. DI & Coupling

9/10
All UseCases, Cubits, Datasources receive dependencies via constructor (abstract types)
No C.* or W.* global accessors remain — all refactored to typed locator getters backed by abstract interfaces
locator<T>() convenience getters used in core/widgets/* and some pages (ShowcaseService) — mockable via GetIt but slightly couples to service locator pattern

3. State Management Consistency

10/10
100% Cubit-only — zero GetX remnants in lib/ and test/
All states use @freezed sealed classes
Scoped BlocProviders at route/page level for feature-specific cubits
Global MultiBlocProvider only retains app-wide cubits (Dashboard, Home, GetSurah, Bookmark)

4. Error Handling Strategy

10/10
Repository returns Future<Either<Failure, T>> via fpdart
DataException at datasource, mapped to domain Failure at repository
GPS retry with locationError/locationPermissionError states after 3 attempts
main.dart error boundaries route through locator<CrashReporter>() — only setCrashlyticsCollectionEnabled is direct (platform init, cannot be abstracted)

5. Navigation & Routing

10/10

int.tryParse for all path params with redirect to /home + CrashReporter logging on invalid
Redirect guard extracted to pure testable evaluateRedirect() method
